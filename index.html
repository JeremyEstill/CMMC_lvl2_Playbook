Phase 0

Set Expectations & Assign Roles

Before touching technology, know the mission. Success is not "having a nice SSP." It is a fully implemented, evidenced, and assessed program — with clearly assigned ownership at every layer.

What success actually looks like: CUI is tracked and protected. The assessment scope is accurate and defensible. All 110 NIST SP 800-171 requirements are implemented with evidence. The correct Level 2 assessment path is completed, including affirmation and SPRS reporting.

Phase 1

Identify What You Are Protecting

This is where most organizations win or lose. Everything downstream — scope, SSP, controls, evidence — depends on a precise understanding of where CUI exists and how it moves.

Step 1 — Confirm CUI Presence

  • Do we receive CUI from DoD or prime contractors?
  • Do we create CUI while performing on contract?
  • Is CUI properly marked and identifiable?
  • Where is it stored, emailed, shared, backed up, printed, or discussed?

Note: NIST SP 800-171 applies to protecting the confidentiality of CUI in nonfederal systems. If no CUI is present, Level 2 may not be the applicable target for that environment.

Step 2 — Build the CUI Data Flow

Map CUI through its full lifecycle: Enters → Processed → Stored → Shared → Archived / Destroyed. For each stage, identify people, devices, applications, servers, networks, cloud services, physical locations, and external parties.

Required Documents

CUI Identification Register CUI Data Flow Diagram CUI Asset Inventory CUI Use Case Matrix External Connections Register Third-Party / Subcontractor CUI Register Boundary Diagram / Network Diagram
Phase 2

Define the Assessment Scope

This is the most critical decision point in the program. If scope is wrong, every control, policy, diagram, and assessment answer will be wrong. The official Level 2 scoping guidance must be followed before any assessment scope is finalized.

If we scope badly, every control, every policy, every diagram, and every assessment answer will be wrong.

Minimum Scope Inclusions

  • Assets that process, store, or transmit CUI
  • Assets that provide security protection for CUI assets
  • Relevant users, administrators, and support personnel
  • Relevant physical spaces
  • Relevant external connections and service providers

Strategic Scoping Decision

Option A
Scope the Whole Enterprise

Best when the organization is already mature and most business systems touch CUI. Broader scope means broader assessment coverage.

Option B — Recommended for Speed
Build a Dedicated CUI Enclave

Best for organizations pursuing Level 2 quickly. A well-designed enclave is easier to defend, document, and assess than a flat enterprise environment.

Required Documents

CMMC Scope Statement Assessment Boundary Diagram Authorized Asset Inventory Security Protection Asset List User / Role Inventory Physical Scope Statement External Service Provider Matrix Inherited Controls Matrix Shared Responsibility Matrix
Phase 3

Choose Your Compliance Path

Level 2 compliance may be achieved through self-assessment or third-party certification, depending on contract requirements. Understand both paths — and build to the higher standard regardless.

Path A
Level 2 Self-Assessment
  1. Conduct internal assessment
  2. Document results
  3. Submit results in SPRS
  4. Submit affirmation in SPRS
  5. Repeat within three years to maintain compliance
Path B
Level 2 C3PAO Certification
  1. Engage an authorized / accredited C3PAO
  2. Provide scope, SSP, policies, diagrams, and evidence
  3. Support interviews, examination, and testing
  4. Receive MET / NOT MET results
  5. Resolve any allowable POA&M items within 180 days
  6. Submit required affirmation in SPRS
Governing principle: Build as if a C3PAO will examine everything — even when the contract only requires self-assessment. That mindset prevents sloppy documentation and weak evidence, and eliminates costly rework if certification is required later.

Required Documents

Assessment Strategy Memo Contract Requirement Traceability Matrix SPRS Submission Plan C3PAO Readiness Plan (if certification required)

Phase 5

Build the System Security Plan (SSP)

The SSP is the story of the environment. If the story is vague, the assessor assumes the control is weak. Every control section must be defensible on its own.

Required SSP Content

Name, purpose, business function, owner, authorization context
Scope & Boundary
Boundary description, network architecture, data flow summary
Asset Inventory
Devices, applications, locations, users, roles
Dependencies
External connections, inherited controls, shared responsibilities
Control Statements
All 110 requirements with implementation details and evidence references
Gaps & Monitoring
POA&M links, continuous monitoring approach, known exceptions

Per-Control Section Structure

For each of the 110 requirements, document: Requirement → Applicability → Implementation Statement → Responsible Role → Technology/Mechanism → Frequency → Evidence References → Exceptions → Associated Procedure → Assessment Note. This structure maps directly to the examine/interview/test assessment methods.

Phase 8

Perform the Internal Self-Assessment

Even if a C3PAO assessment is scheduled, conduct a brutally honest internal assessment first. This surfaces gaps while you still have time to remediate — not during an external review.

Do not allow "sort of implemented" to be scored as MET. If a procedure says something happens but there is no proof it happened, that is a gap — treat it as such until proven otherwise.

Assessment Approach

Apply the official NIST SP 800-171A assessment objectives. For each requirement, evaluate using examine, interview, and test methods. Determine objectively whether evidence shows the requirement is met.

Internal Assessment Outputs

Internal Assessment Plan Requirement-by-Requirement Assessment Workbook Interview Notes Test Results Log Deficiency Log Draft Score Sheet Management Review Brief
Phase 9

Build & Manage the POA&M

A POA&M is not a wish list. It is a disciplined closure plan. CMMC rules sharply limit what can sit on a POA&M — and for how long.

The organization must score MET on at least 80% of Level 2 requirements to be eligible for a conditional status. Certain requirements — including the SSP itself — cannot be placed on a POA&M under any circumstances. For C3PAO conditional status, POA&M closeout must occur within 180 days.

Phase 10

Management Review & Readiness Decision

Before any external submission or certification engagement, conduct a formal leadership readiness review. Assessment results and affirmations carry legal and contractual consequences.

Readiness Gate Questions

  • Is the assessment scope final and defensible?
  • Is the SSP current, complete, and approved?
  • Is each requirement marked MET or on an allowable POA&M?
  • Are all interview owners assigned and prepared?
  • Are external service provider responsibilities fully documented?
  • Are all diagrams and inventories current and accurate?
  • Have we identified and resolved all shadow IT or undocumented systems?
  • Are we prepared to truthfully affirm compliance?

Required Governance Documents

Readiness Review Deck Executive Risk Acceptance Memo Assessment Go / No-Go Memo Final Evidence Freeze Notice SSP Approval Memo
Phase 11

Submit or Certify

There are two finish lines, determined by contract requirement. Both paths require documented results and a formal affirmation submitted in SPRS.

Path A — Self-Assessment
Level 2 Self-Assessment
  1. Conduct the self-assessment using 800-171A objectives
  2. Document all results
  3. Submit results to SPRS
  4. Submit affirmation in SPRS
  5. Maintain and repeat within three years
Path B — Certification
Level 2 C3PAO Certification
  1. Engage an authorized / accredited C3PAO
  2. Provide all scoping and evidence packages
  3. Support all interviews, examinations, and tests
  4. Receive official MET / NOT MET results
  5. Resolve eligible POA&M items within 180 days if conditional
  6. C3PAO submits results to CMMC eMASS → SPRS
  7. Organization submits affirmation in SPRS

Governing Principles

Five Rules to Live By

These principles govern every decision, document, and implementation choice throughout the program lifecycle.

1
Follow the CUI
Never start with tools. Start with where CUI exists and how it moves. That drives scope — and scope drives everything else.
2
If it is not documented, it is not mature
A control without policy, procedure, and evidence is fragile. The assessment process is built entirely around objective evidence.
3
If the scope changes, the SSP changes
Every major architecture change triggers SSP, diagram, inventory, and evidence updates. Security is part of the system lifecycle — not a one-time event.
4
A POA&M is limited
Do not rely on "we will fix it later." CMMC rules sharply constrain what can sit on a POA&M and for how long. Deficiencies must be resolved, not deferred indefinitely.
5
Build for C3PAO — even if you start with self-assessment
That mindset raises evidence quality and eliminates rework. DoD uses one Level 2 assessment framework for both paths.
Execution Reference

Practical Execution Sequence

A recommended implementation order for organizations executing this program in a real-world environment.

Weeks 1–2
Identify contracts, CUI, owners, and scope candidates. Make the enclave vs. enterprise decision.
Next
Create diagrams, asset inventories, external connection register, and provider list. Draft the scope statement.
Then
Draft the SSP skeleton and all core policies and procedures across all 14 families.
Then
Implement technical and administrative controls, working family by family against the requirement list.
Then
Collect evidence into a structured repository aligned to the 14-folder binder structure.
Then
Perform the internal assessment using NIST SP 800-171A examine / interview / test methodology.
Then
Build the POA&M for all identified gaps. Begin remediation immediately.
Then
Conduct the formal management readiness review. Resolve all go / no-go items.
Finally
Complete SPRS self-assessment affirmation or engage a C3PAO for certification — based on contract requirement.
Ongoing
Maintain compliance through continuous monitoring, scheduled reviews, and timely SSP updates.
